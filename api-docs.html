StockMonitor API 文档 v1

股票实时监控系统 REST API · 前后端分离架构 · JWT 认证
基础信息 认证接口 股票管理 分组管理 预警系统 行情数据 监控控制 错误码
⚠️ 认证说明：除登录/注册外的所有接口均需携带 JWT Token，格式为 Authorization: Bearer <token>。Token 有效期 24 小时。

基础信息

项目说明
Base URLhttp://localhost:5000/api/v1
认证方式JWT Bearer Token (HS256)
请求格式application/json
响应格式application/json
CORS已启用 (支持跨域)

统一响应格式：

// 成功
{"code": 0, "msg": "ok", "data": {...}}

// 失败  
{"code": 40101, "msg": "未提供认证令牌", "error": "UNAUTHORIZED"}

认证接口

/api/v1/auth/*

POST/auth/register — 用户注册

参数说明必填
username用户名 (≥3字符)是
password密码 (≥6字符)是
// Response
{"code":0,"msg":"注册成功","data":{"user_id":1,"username":"admin","token":"eyJ...","expires_at":"..."}}

POST/auth/login — 用户登录

参数说明必填
username用户名是
password密码是
// Response
{"code":0,"msg":"登录成功","data":{"token":"eyJ...","token_type":"Bearer","expires_at":"..."}}

POST/auth/logout 需认证 — 登出 (Token 加入黑名单)

curl -X POST http://localhost:5000/api/v1/auth/logout \
  -H "Authorization: Bearer TOKEN"

GET/auth/me 需认证 — 获取当前用户信息

// Response
{"code":0,"data":{"user_id":1,"username":"admin","role":"user","created_at":"...","last_login":"..."}}

股票管理

/api/v1/stocks/*

GET/stocks 需认证 — 获取所有股票

// Response
{"code":0,"data":[{"code":"600519","name":"贵州茅台","market":"A","added_at":"...","is_active":1}]}

GET/stocks/search?keyword=茅台 需认证 — 搜索股票

// Response  
{"code":0,"data":[{"code":"600519","name":"贵州茅台","market":"SH"}]}

POST/stocks 需认证 — 添加股票

参数说明必填
code股票代码是
name股票名称是
market市场 (默认 A)否

DELETE/stocks/{code} 需认证 — 删除股票

PUT/stocks/{code}/tags 需认证 — 更新单只股票标签

tags标签字符串（逗号分隔）是

POST/stocks/retag 需认证 — 一键为所有股票重新打标签

分组管理

/api/v1/groups/*

GET/groups 需认证 — 获取所有分组

POST/groups 需认证 — 创建分组

name分组名称是
color颜色 (默认 #3b82f6)否

PUT/groups/{id} 需认证 — 更新分组

DELETE/groups/{id} 需认证 — 删除分组

POST/groups/{id}/stocks 需认证 — 股票加入分组

DELETE/groups/{id}/stocks/{code} 需认证 — 从分组移除

预警系统

/api/v1/alerts/*

GET/alerts/rules 需认证 — 获取预警规则

POST/alerts/rules 需认证 — 创建预警规则

name规则名称是
rule_type类型 (见下表)是
params参数对象 {threshold: 5}是
stock_codes股票代码数组是
notify_feishu飞书通知 (0/1)否

支持的预警类型：

rule_type说明参数
price_up价格向上突破threshold (价格阈值)
price_down价格向下跌破threshold
change_up涨幅超过阈值threshold (%)
change_down跌幅超过阈值threshold (%)
volume_surge成交量异动multiplier, avg_days
turnover_high换手率过高threshold (%)
amplitude_high振幅过大threshold (%)
price_break_ma价格突破均线direction, ma_period
continuous_up连续上涨days
continuous_down连续下跌days
volume_ratio量比异常threshold
limit_up触及涨停—
limit_down触及跌停—

GET/alerts/logs?limit=50 需认证 — 获取预警日志

POST/alerts/logs/{id}/read 需认证 — 标记已读

POST/alerts/logs/read-all 需认证 — 一键标记全部已读
